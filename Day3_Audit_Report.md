## 1. Blogy Dashboard UX Audit

* **High Barrier to Entry (Registration Gatekeeping):** The platform requires a live top-level domain (TLD) to even complete registration. This creates significant friction for developers and students in the prototyping phase who do not yet have a hosted site.
* **Onboarding Dead-Ends:** The strict domain validation creates an immediate barrier for users attempting to audit the tool's SEO capabilities before committing to a full deployment.
* **UX Feedback Gap:** Absence of asynchronous progress indicators (loading spinners/bars) during heavy data-processing tasks such as keyword clustering or article generation.

## 2. Complex API Setups

* **Lack of Sandbox/Localhost Support:** There is no 'Trial' or 'Localhost' mode available for API configuration. This prevents a Data Strategist from testing the handoff contract or verifying API connectivity without a production-ready domain.
* **Integration Scavenger Hunt:** Lack of contextual documentation links during the API handshake process, requiring users to manually search for external CMS token settings.

## 3. Generative Engine Optimization (GEO) Proposal

### GEO Scoring Framework
To calculate the final $GEO_{score}$, each metric is graded on a scale of 1 to 10:

* **$Score_{Structure}$ (15%):** Use of H1-H4 tags and schema markup.
* **$Score_{Semantic}$ (25%):** Presence of LSI terms and keyword density.
* **$Score_{Interpretability}$ (30%):** Ease of LLM summarization.
* **$Score_{Conversational}$ (20%):** Natural language patterns and Direct Answer Optimization.
* **$Score_{Engagement}$ (10%):** Internal linking and CTA placement.

**Final formula:**
$$
GEO_{score} = \frac{(Score_{Structure} \times 0.15) + (Score_{Semantic} \times 0.25) + (Score_{Interpretability} \times 0.30) + (Score_{Conversational} \times 0.20) + (Score_{Engagement} \times 0.10)}{5} \times 100
$$
